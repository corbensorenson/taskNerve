# Governance

## Model

fugit-alpha is open source (MIT) and maintainer-led.

- The repository owner/maintainer controls upstream roadmap, merges, release timing, and project direction for the mainline repository.
- Contributions are welcome through issues and pull requests.
- Forks are allowed under the MIT license.

## Scope

This governance file defines how the **upstream project** is operated.

It does not restrict rights granted by the MIT license for downstream users/forks.

## Decision Process

- Small fixes: maintainer may merge directly.
- Feature changes: maintainer decides acceptance, sequencing, and release scope.
- Breaking changes: should include migration notes in README/CHANGELOG before release.

## Release Authority

Only the maintainer controls official releases and release tags for upstream.
