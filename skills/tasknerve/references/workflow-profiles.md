# Archived Reference

This reference file described the deprecated CLI-era TaskNerve workflow.

On `codex/codex-native`, the supported model is:
- TaskNerve settings are managed from the native TaskNerve page inside Codex
- day-to-day queue work happens through the task drawer
- controller and worker orchestration runs through Codex-native threads and inference
- project contracts live in `project_goals.md`, `project_manifest.md`, and `contributing ideas.md`
